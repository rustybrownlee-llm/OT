# SOW-017.0 - Brownfield Wastewater Device and Network Atoms

**Status**: Approved
**Created**: 2026-03-02
**Priority**: High
**Dependencies**: SOW-016.0 (Environment Archetype Schema Extensions)
**OT Domain Review**: Complete (20 corrections applied, 5 improvements incorporated)
**ADR Alignment Check**: Complete (all 6 ADRs ALIGNED, no conflicts)

## 1. Objective

Create the device atoms and network atoms needed for the brownfield wastewater treatment facility -- the Frankenstein hybrid environment that proves composability. Reuse existing device atoms (SLC-500, CompactLogix, Moxa NPort) with new wastewater-specific register map variants. Create one new device atom (Cradlepoint IBR600 cellular modem) for the vendor remote access gateway. Create four new network atoms modeling the hybrid topology (flat Ethernet, DH-485 serial bus, Level 3 VLAN, cellular WAN). Deliver a facility narrative document establishing the 25-year operational history that drives era markers and teaching points.

## 2. Background

The brownfield wastewater treatment plant (milestone-b0.4.md) models a municipal facility built in 1997 with ad-hoc modernization through 2022. Per ADR-010 D6, this Frankenstein hybrid must demonstrate that a complex, multi-era environment can be expressed entirely in atomic YAML without plant code changes.

The facility has five distinct eras of technology:

| Era | Year | Technology Added |
|-----|------|-----------------|
| Original build | 1997 | 2x SLC-500 on DH-485 serial bus, flat Ethernet, unmanaged switch |
| Ethernet bridge | 2008 | Moxa NPort gateway bridging serial to Ethernet |
| Partial modernization | 2013 | CompactLogix for new aeration system, connected to flat Ethernet |
| Audit response | 2018 | Managed switch, VLAN 30 for SCADA server (Level 3 only) |
| Vendor remote access | 2022 | Cradlepoint IBR600 cellular modem on flat network |

This SOW delivers the atomic building blocks. SOW-018.0 composes them into the environment definition with placements, era markers, and boundary states.

### Atom Reuse Analysis

Three of five device types already have atoms in the design library:

| Device | Existing Atom | Reuse Strategy |
|--------|--------------|----------------|
| SLC-500/05 | `slc-500-05.yaml` | Add 2 register map variants (`ww-influent`, `ww-effluent`) |
| CompactLogix L33ER | `compactlogix-l33er.yaml` | Add 1 register map variant (`ww-aeration`) |
| Moxa NPort 5150 | `moxa-nport-5150.yaml` | Add 1 register map variant (`ww-serial-gateway`) |
| Cradlepoint IBR600 | None | New device atom required |

The managed switch decision resolves milestone-b0.4.md open question #2: modeling as a network atom property is simpler and consistent with existing patterns (wt-level1 already uses `managed_switch: true`). The topology visualization (SOW-019.0) can derive switch type from the network atom. A dedicated infrastructure device atom is deferred as future enhancement.

## 4. Requirements

### Functional Requirements

1. FR-001: The `slc-500-05.yaml` device atom SHALL have two new register map variants: `ww-influent` (influent screening and primary treatment) and `ww-effluent` (secondary treatment and effluent discharge).
2. FR-002: The `ww-influent` variant SHALL model a 1997-era SLC-500 controlling bar screening, grit removal, and primary clarification. Registers SHALL include influent flow rate, influent flow totalizer (writable for daily reset -- permit compliance attack surface) [OT-REVIEW: IMPROVEMENT-2], bar screen differential pressure (0.0-12.0 in WC) [OT-REVIEW: CORRECTION-1], grit chamber level, primary clarifier level (0.0-5.0 ft, relative to weir elevation) [OT-REVIEW: CORRECTION-5], sludge blanket depth, sludge pump cycle time (writable, 0-120 min -- attack surface for slow-burn primary clarifier carryover) [OT-REVIEW: CORRECTION-4], influent pH (0.0-14.0 transmitter full scale; normal 6.5-7.5) [OT-REVIEW: CORRECTION-2], and influent temperature (40-90 degF; wastewater never reaches 32 degF due to plumbing thermal buffering) [OT-REVIEW: CORRECTION-3]. Coils SHALL include bar screen motor, grit pump, primary sludge pump, and influent gate. All holding registers stored as 16-bit scaled integers per SLC-500 ProSoft module convention (N7: integer files mapped to Modbus registers, not IEEE 754 floats) [OT-REVIEW: IMPROVEMENT-3].
3. FR-003: The `ww-effluent` variant SHALL model a 1997-era SLC-500 controlling secondary clarification and effluent discharge. Registers SHALL include effluent flow rate, secondary clarifier level (0.0-5.0 ft, relative to weir elevation) [OT-REVIEW: CORRECTION-5], return activated sludge (RAS) flow, waste activated sludge (WAS) flow (0-100 GPM; 500 GPM is 5x too high for a 5 MGD plant) [OT-REVIEW: CORRECTION-6], effluent TSS, effluent ammonia nitrogen (0.0-30.0 mg/L, online ion-selective electrode -- replaces BOD which is a 5-day lab test, not a real-time PLC register) [OT-REVIEW: CORRECTION-7], UV transmittance, and permit interlock status (read-only bitmask: bit0=UV_system_active, bit1=flow_within_permit, bit2=NPDES_permit_active -- multi-step attack requires spoofing interlock AND valve) [OT-REVIEW: CORRECTION-8]. Coils SHALL include RAS pump, WAS pump, UV disinfection enable, and effluent discharge valve.
4. FR-004: The `compactlogix-l33er.yaml` device atom SHALL have one new register map variant: `ww-aeration` (aeration blower control, 2013 modernization).
5. FR-005: The `ww-aeration` variant SHALL model dissolved oxygen control via variable-frequency blowers. Registers SHALL include blower speed setpoint (0-100%, writable VFD command) and blower speed feedback (0-3600 RPM, read-only shaft speed from VFD) [OT-REVIEW: CORRECTION-9], DO setpoint (0.0-8.0 mg/L, writable -- primary attack surface for aeration: writing 0.0-0.5 creates anoxic conditions) [OT-REVIEW: CORRECTION-10], dissolved oxygen measurement (0.0-12.0 mg/L, read-only), air flow rate, blower discharge pressure, blower inlet temperature, blower motor current, aeration basin level (0.0-15.0 ft) [OT-REVIEW: CORRECTION-12], mixed liquor suspended solids (MLSS), specific oxygen uptake rate (SOUR -- PLC-calculated from DO uptake rate and MLSS, not a raw sensor; near-zero indicates dead sludge) [OT-REVIEW: CORRECTION-11, IMPROVEMENT-5], and blower run hours (read-only, predictive maintenance -- this is what the vendor reads via Cradlepoint) [OT-REVIEW: IMPROVEMENT-1]. Coils SHALL include blower 1 run, blower 2 run, aeration valve auto, and DO control enable.

## 5. Design

### Register Map Design: Wastewater Process Context

**Influent Treatment** (SLC-500 `ww-influent`):
The influent side handles raw wastewater arriving at the plant. Bar screens remove large debris. Grit chambers settle out sand and gravel. Primary clarifiers allow solids to settle, producing primary sludge. The SLC-500 was installed in 1997 to automate pump sequencing and level control. It communicates via DH-485 through the Moxa gateway (2008).

Realistic parameters [OT-REVIEW corrected]:
- Influent flow: 0-5000 GPM (typical for small municipal WWTP, 2-7 MGD capacity)
- Influent flow totalizer: 0-99999 gallons x1000 (writable -- operators reset at midnight for daily NPDES reporting; spoofing is a permit compliance attack) [OT-REVIEW: IMPROVEMENT-2]
- Bar screen differential: 0.0-12.0 in WC (triggers automatic rake above ~6 in WC; 12 in WC = plugged screen alarm) [OT-REVIEW: CORRECTION-1]
- Grit chamber level: 0-100% (triggers grit pump above 80%)
- Primary clarifier level: 0.0-5.0 feet (relative to weir elevation, not tank depth; normal variation <1 ft) [OT-REVIEW: CORRECTION-5]
- Sludge blanket depth: 0.0-8.0 feet (operator monitors for rising blanket)
- Sludge pump cycle time: 0-120 minutes (writable setpoint; writing 0 stops sludge withdrawal -- slow-burn carryover attack) [OT-REVIEW: CORRECTION-4]
- pH: 0.0-14.0 (transmitter full scale; normal influent 6.5-7.5; below 5.5 or above 9.5 indicates industrial discharge; full scale enables spoofed extreme values to display correctly) [OT-REVIEW: CORRECTION-2]
- Temperature: 40-90 degF (wastewater never reaches 32 degF due to building plumbing thermal buffering; below 55 degF impairs downstream biological treatment) [OT-REVIEW: CORRECTION-3]

All SLC-500 registers stored as 16-bit scaled integers per ProSoft MVI46-MCM convention (N7: integer files mapped to Modbus registers, not IEEE 754 floats). Scale_min/scale_max define the engineering unit interpretation of the 0-32767 raw integer range. [OT-REVIEW: IMPROVEMENT-3]

**Effluent Treatment** (SLC-500 `ww-effluent`):
The effluent side handles treated water before discharge. Secondary clarifiers separate activated sludge from treated effluent. RAS (return activated sludge) pumps maintain the biological process. WAS (waste activated sludge) pumps remove excess biomass. UV disinfection inactivates pathogens. The effluent discharge valve is the final control before the outfall.

Realistic parameters [OT-REVIEW corrected]:
- Effluent flow: 0-5000 GPM (matches influent capacity)
- Secondary clarifier level: 0.0-5.0 feet (relative to weir elevation, not absolute tank depth) [OT-REVIEW: CORRECTION-5]
- RAS flow: 0-2500 GPM (typically 25-75% of influent flow)
- WAS flow: 0-100 GPM (small fraction -- typically 2-30 GPM for a 5 MGD plant; 500 GPM would waste the entire sludge inventory in hours) [OT-REVIEW: CORRECTION-6]
- TSS: 0.0-50.0 mg/L (NPDES permit typically requires <30 mg/L)
- Ammonia nitrogen: 0.0-30.0 mg/L (online ion-selective electrode analyzer; replaces BOD which is a 5-day lab test that cannot be measured in real time by any PLC) [OT-REVIEW: CORRECTION-7]
- UV transmittance: 0-100% (effluent clarity indicator; below 55% means poor treatment)
- Permit interlock status: bitmask (bit0=UV_system_active, bit1=flow_within_permit, bit2=NPDES_permit_active; read-only -- plant safety requires all bits set before discharge is permitted) [OT-REVIEW: CORRECTION-8]

**Attack surface teaching point**: The effluent discharge valve has a permit interlock: the valve cannot be opened unless the UV disinfection system is operational and effluent meets quality standards. A single-register attack on the valve coil is insufficient -- an attacker must also spoof the interlock register or disable the UV system. This multi-step attack is more realistic than the common IT assumption of "just flip a bit." Writing to the WAS pump coil with even 50-100 GPM uncontrolled wasting over 24 hours destroys the activated sludge inventory -- recovery takes days to weeks. These are regulatory and environmental consequences (significant fines, public health risk), not safety-of-life. [OT-REVIEW: CORRECTION-8]

**Aeration** (CompactLogix `ww-aeration`):
The aeration system was the 2013 modernization project. Variable-frequency blowers inject air into the aeration basin to maintain dissolved oxygen (DO) for the activated sludge biological process. The CompactLogix L33ER was selected because the 1997 SLC-500s lacked the I/O density and Ethernet connectivity needed for VFD communication.

Realistic parameters [OT-REVIEW corrected]:
- Blower speed setpoint: 0-100% (writable VFD command -- what the PLC writes to the drive) [OT-REVIEW: CORRECTION-9]
- Blower speed feedback: 0-3600 RPM (read-only -- actual shaft speed reported from VFD) [OT-REVIEW: CORRECTION-9]
- DO setpoint: 0.0-8.0 mg/L (writable -- PRIMARY ATTACK SURFACE: writing 0.0-0.5 mg/L creates anoxic conditions; the PLC's PID loop will obediently reduce blower speed to match the spoofed setpoint) [OT-REVIEW: CORRECTION-10]
- Dissolved oxygen measurement: 0.0-12.0 mg/L (read-only probe measurement; control target typically 2.0 mg/L; below 1.0 = dead zone, above 4.0 = energy waste)
- Air flow: 0-5000 SCFM (standard cubic feet per minute)
- Discharge pressure: 0.0-15.0 PSI (blower output pressure)
- Inlet temperature: 30-120 degF (ambient to heated)
- Motor current: 0.0-200.0 amps (blower motor load indicator)
- Basin level: 0.0-15.0 feet (typical municipal aeration basin SWD is 10-15 ft) [OT-REVIEW: CORRECTION-12]
- MLSS: 0-10000 mg/L (mixed liquor suspended solids; target 2000-4000 mg/L)
- SOUR: 0.0-30.0 mg O2/g MLSS/hr (PLC-calculated from DO uptake rate and MLSS measurement, not a raw sensor input; near-zero indicates dead or severely inhibited sludge -- biological crash; cannot be spoofed by replaying a Modbus write without first spoofing the source inputs) [OT-REVIEW: CORRECTION-11, IMPROVEMENT-5]
- Blower run hours: 0-999999 hours (read-only, predictive maintenance counter; this is what the blower vendor reads via the Cradlepoint cellular link) [OT-REVIEW: IMPROVEMENT-1]

**Attack surface teaching point**: The DO setpoint register is the canonical aeration attack. Writing a DO setpoint of 0.5 mg/L via unauthenticated Modbus while DO control is enabled causes the PLC to reduce blower speed to minimum, the basin goes anoxic, and the biomass crashes. Recovery takes days to weeks. Alternatively, writing 100% blower speed setpoint while DO is already at 8 mg/L wastes massive energy (blowers are 50-70% of a WWTP's electricity cost). The VFD setpoint/feedback split teaches students that PLCs don't directly control motor RPM -- they write a percentage command to the VFD, which executes it. [OT-REVIEW: CORRECTION-9, CORRECTION-10]

**Serial Gateway** (Moxa `ww-serial-gateway`):
Installed in 2008 to enable remote monitoring from city hall without replacing the DH-485 infrastructure. The Moxa NPort 5150 bridges serial and Ethernet, making the SLC-500s accessible via Modbus TCP. The register variant mirrors the existing `serial-gateway` variant (connection status, serial port config, message counts) with wastewater-specific descriptions.

**Cellular Modem** (Cradlepoint IBR600 `vendor-remote-access`):
Installed in 2022 by the blower vendor for predictive maintenance access. Connected to the flat network (192.168.10.0/24) because no one asked about network architecture. Default credentials were never changed. NAT is enabled, providing the vendor cloud-based access to the entire flat OT network.

The register map is an educational abstraction: real Cradlepoint devices expose management via HTTPS/SNMP, not Modbus. We model management status as Modbus TCP registers so the monitoring module can discover and poll it alongside real PLCs. This teaches engineers that a cellular gateway is visible as a device on the OT network -- and its presence is often a surprise during network discovery exercises.

Registers [OT-REVIEW corrected]:
- WAN link status (0=down, 1=connecting, 2=connected)
- Signal strength: 0-120 unsigned magnitude (0=no signal/-120 dBm, 120=perfect/0 dBm; typical LTE good: <60, poor: >90; stored as unsigned to avoid 16-bit signed Modbus encoding issues) [OT-REVIEW: CORRECTION-15]
- Data usage: 0-999 GB (fits 16-bit register; far exceeds any maintenance cellular data plan) [OT-REVIEW: CORRECTION-16]
- Uptime: 0-65535 seconds (16-bit register, wraps at ~18 hours; acceptable for simulation)
- Active LAN clients: 0-10 (actual OT device count on static-IP network, not DHCP scope -- OT devices use static IPs; typical value 2-5) [OT-REVIEW: CORRECTION-17]
- NAT mode: 0=disabled, 1=enabled (always 1 in default config)
- VPN tunnel status: 0=none, 1=configured, 2=active

### Network Atom Design

**ww-flat** (1997 Original Flat Segment):
192.168.10.0/24 -- the original construction Ethernet backbone. Unmanaged 24-port switch (Netgear or similar commodity hardware). All Level 1 devices connect here: SLC-500s (via Moxa), CompactLogix (2013 addition), Cradlepoint (2022 addition). This is the segment with the largest attack surface: internet-connected (via Cradlepoint), flat (no segmentation from Level 3), and unmonitored (no SPAN capability).

**ww-serial-bus** (1997 DH-485 Backbone):
The original DH-485 multidrop serial bus connecting the SLC-500 PLCs. Cannot be retired without full plant shutdown. Only accessible via the Moxa NPort gateway (2008). Devices on this bus are invisible to IP-based tools. Models the serial backbone persistence pattern described in milestone-b0.4.md.

**ww-level3** (2018 Compliance VLAN):
10.30.30.0/24, VLAN 30 -- added during the 2018 state compliance audit response. Managed switch (Cisco Catalyst 2960 or similar) with SPAN capability. Only the SCADA server sits on this segment. No simulated devices are placed on ww-level3 in SOW-018.0 (SCADA server is out of scope per TD-030). The network atom exists to express the boundary state `enforced` between Level 3 and the flat segment, enabling the topology visualization (SOW-019.0) to render the managed switch boundary correctly. [OT-REVIEW: CORRECTION-19]

Teaching point: The managed switch and VLAN create a real boundary -- but only for the SCADA server. Everything else (PLCs, gateway, cellular modem) remains on the flat segment. The audit checkbox was checked, but the actual attack surface reduction is minimal. An attacker on the flat segment can reach 5 of 6 devices; only the SCADA server is protected.

**ww-wan** (2022 Cellular Uplink):
The Cradlepoint's cellular WAN connection. Point-to-point to the carrier network (and from there, to the vendor's cloud platform). Modeled as `ethernet` type (logical framing) with a /30 subnet. This is the internet exposure vector described in milestone-b0.4.md. Traffic on this link is invisible to all on-premises monitoring tools -- no SPAN, no local packet capture, no IDS visibility. An attacker or vendor using this path creates traffic that bypasses all local detection entirely. This is a fundamentally different blind spot from the unmanaged switch on ww-flat (where traffic at least exists on the local wire, even if no SPAN port mirrors it). [OT-REVIEW: CORRECTION-20]

## 10. Risks and Mitigations

| Risk | Impact | Probability | Mitigation |
|------|--------|-------------|------------|
| Wastewater register values may be unrealistic | Medium | Medium | OT domain review validates ranges against typical municipal WWTP parameters. Scale ranges documented with source rationale. |
| Cradlepoint Modbus TCP abstraction may confuse learners who check real specs | Low | Medium | Device atom description explicitly states this is an educational abstraction. Real Cradlepoints use HTTPS/SNMP for management. |
| Register map variants may not exercise sufficient attack surface | Medium | Low | Each variant includes writable registers (setpoints, pump commands, valve controls). FR-019 requires attack surface annotation. |
| Too many registers per variant creates implementation burden in SOW-018.0 | Low | Low | Variant sizes are consistent with existing atoms (7-9 holding + 3-4 coils). Plant runtime already handles this scale. |
| DH-485 vs RS-485 confusion in serial bus network atom | Medium | Medium | Network atom type is `serial-rs485` (electrical standard). Comments clarify DH-485 is the protocol running on RS-485 physical layer, consistent with existing `mfg-serial-bus.yaml` and OT domain review lesson from SOW-001.0. |
