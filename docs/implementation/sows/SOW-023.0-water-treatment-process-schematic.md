# SOW-023.0 - Water Treatment Process Schematic

**Status**: Approved
**Created**: 2026-03-02
**Priority**: High
**Dependencies**: SOW-021.0 (Process Schematic Schema and Validation)

## 1. Objective

Author the `process.yaml` for the greenfield water treatment environment, mapping every physical equipment item and instrument to its corresponding Modbus register. This is the first real process schematic -- it proves the schema works with actual register data and produces a validated artifact ready for SVG rendering (SOW-025.0).

## 2. Background

The water treatment facility has three CompactLogix PLCs controlling three stages of the treatment process: intake, treatment, and distribution. Together they expose 17 holding registers and 11 coils representing sensors, actuators, and status points across the physical process.

The process.yaml must map each register to a piece of physical equipment using ISA-5.1 instrument tags, providing the bridge between "register address 2 on port 5021" and "sand filter differential pressure is 12.3 kPa." This is the core educational value of Beta 0.5.

**[OT-REVIEW] Critical tagging principle**: ISA-5.1 distinguishes transmitters (read-only sensor measurements, suffix "T") from controllers (writable command outputs, suffix "C" or "IC"). Writable registers MUST use controller-type tags (SC, FIC, etc.), NOT transmitter-type tags (ST, FT, etc.). Teaching trainees that a writable setpoint is a "transmitter" contradicts the fundamental OT concept that sensors are read-only and actuators are writable.

## 5. Design

### Physical Equipment to Register Mapping

**[OT-REVIEW] Tagging principle applied**: Writable registers (setpoints, commands) use controller suffixes (SC, FIC). Read-only registers (measurements, feedback) use transmitter suffixes (FT, PT, AT, etc.). This teaches the fundamental ISA-5.1 distinction between measured variables and control outputs.

| Equipment | Type | Instruments | Register |
|-----------|------|-------------|----------|
| Travelling Screen | screen | ZS-101 screen_wash_active | Coil 2 (read-only status) |
| Raw Water Intake Basin | basin | FT-101 intake_flow_rate | HR 0 (L/s, 0-100, read-only) |
| | | AT-101 raw_water_ph | HR 2 (pH, 0-14, read-only) |
| | | AT-102 raw_water_turbidity | HR 3 (NTU, 0-100, read-only) |
| | | TT-101 intake_water_temp | HR 4 (degC, 0-40, read-only) |
| | | LS-101 low_well_level_alarm | Coil 3 (read-only discrete level alarm) |
| Intake Pump 1 | pump | run-P-101 intake_pump_01_run | Coil 0 (writable) |
| | | SC-101 intake_pump_speed | HR 1 (%, 0-100, writable setpoint) |
| Intake Pump 2 | pump | run-P-102 intake_pump_02_run | Coil 1 (writable) |

**[OT-REVIEW corrections applied]**:
- Basin type "basin" (not "tank") -- open intake structure, not a pressure vessel
- ZS-101 (not ZSC-101) -- ZSC means "Position Switch, Closed" which doesn't apply to a wash cycle
- SC-101 (not ST-101) -- writable speed setpoint is a Speed Controller, not a Speed Transmitter
- LS-101 (Level Switch) for low_well_level_alarm -- placed on the basin, not the screen
- FT-101 is correct (read-only flow measurement from inline meter)

| | | PT-202 filter_outlet_pressure | HR 1 (kPa, 0-500, read-only) |
| | | PDT-201 filter_differential_pressure | HR 2 (kPa, 0-50, read-only) |
| | | AT-201 turbidity_post_filter | HR 6 (NTU, 0-5, read-only) |
| | | HS-201 filter_backwash_command | Coil 0 (writable command) |
| | | HS-202 high_dp_alarm | Coil 3 (read-only alarm status) |
| Chemical Feed Pump | pump | FIC-202 chemical_feed_rate | HR 4 (mL/min, 0-500, writable setpoint) |
| | | AT-202 chlorine_residual | HR 5 (mg/L, 0-5, read-only) |
| | | run-P-201 chemical_feed_pump_run | Coil 1 (writable) |
| UV Sterilization | uv_system | RT-203 uv_intensity | HR 3 (mW/cm2, 0-100, read-only) |
| | | run-UV-201 uv_system_active | Coil 2 (read-only status) |

**[OT-REVIEW corrections applied]**:
- FIC-202 (not FT-202) -- writable chemical feed rate is a Flow Indicating Controller, not a Flow Transmitter
- RT-203 (not AT-203) -- UV intensity is radiant power from a photodiode, not an analytical measurement. AT is for water chemistry (pH, turbidity, chlorine). RT (Radiation Transmitter) is correct for UV lamp intensity per ISA-5.1
- AT-202 is correct (read-only chlorine residual measurement downstream of dosing point)
- HS-201 retained for backwash command -- common industry convention for SCADA-initiated backwash even though strictly it is a remote command. Comment in process.yaml documents this convention choice.
- HS-202 added for high_dp_alarm (Coil 3) on the Sand Filter
- Treatment strategy: post-filtration chlorination followed by UV disinfection. Chlorine at the chemical feed point, UV downstream. This order is documented in process.yaml to teach trainees why AT-202 appears before RT-203 in the flow path.

### ISA-5.1 Tag Convention

| Prefix | ISA Meaning | Read/Write | Usage in This Process |
|--------|-------------|------------|----------------------|
| FT | Flow Transmitter | Read-only | intake_flow_rate, distribution_flow_rate |
| FIC | Flow Indicating Controller | Writable | chemical_feed_rate (dosing setpoint) |
| LT | Level Transmitter | Read-only | clear_well_level |
| LS | Level Switch | Read-only | low_well_level_alarm (discrete float switch) |
| PT | Pressure Transmitter | Read-only | filter_inlet_pressure, filter_outlet_pressure, distribution_pressure |
| PDT | Pressure Differential Transmitter | Read-only | filter_differential_pressure |
| TT | Temperature Transmitter | Read-only | intake_water_temp, distribution_water_temp |
| AT | Analyzer Transmitter | Read-only | pH, turbidity, chlorine residual |
| RT | Radiation Transmitter | Read-only | uv_intensity (UV photodiode, not analytical measurement) |
| SC | Speed Controller | Writable | intake_pump_speed (VFD speed setpoint) |
| HS | Hand Switch | Both | filter_backwash_command (convention for SCADA-initiated discrete commands), high_dp_alarm (read-only status) |
| ZS | Position Switch | Read-only | screen_wash_active |
| run | Project extension (NOT ISA-5.1) | Writable | pump/UV run coils |

**[OT-REVIEW] Key distinction**: Transmitter tags (suffix "T") are always read-only sensor measurements. Controller tags (suffix "C" or "IC") are always writable setpoints/commands. This is the fundamental ISA-5.1 principle: the tag tells you whether the point is an input (measurement) or output (command).

### Alarm Thresholds

| Tag | Warning | Alarm | Warning Low | Alarm Low | Rationale |
|-----|---------|-------|-------------|-----------|-----------|
| FT-101 | 80 | 95 | -- | -- | High intake flow overloads treatment capacity |
| AT-102 | 20 | 40 | -- | -- | [OT-REVIEW] High turbidity overloads filtration; most plants intervene by 10-30 NTU |
| PDT-201 | 15 | 25 | -- | -- | Filter fouling triggers backwash at 15-25 kPa |
| AT-201 | -- | -- | 1.0 | 0.5 | [OT-REVIEW corrected] warning_low=1.0 (approaching 1 NTU regulatory limit); alarm_low=0.5 (implausibly low reading, possible sensor failure) |
| RT-203 | -- | -- | 30 | 20 | Low UV intensity risks inadequate disinfection |
| AT-202 | 3.0 | 4.0 | 0.3 | 0.2 | Overdosing (high) vs inadequate disinfection (low) |
| LT-301 | 90 | 95 | 20 | 10 | Overflow risk (high) and dry-run pump damage (low) |
| PT-301 | 600 | 650 | 200 | 100 | Overpressure vs low supply pressure |

**[OT-REVIEW] Threshold ordering rule**: alarm_low < warning_low < normal range < warning < alarm. The alarm must always be more severe than the warning. For low-side thresholds, that means the alarm value is numerically smaller than the warning value.

### Process Flow Path

```
[Travelling Screen] → [Raw Water Basin] → [Intake Pump 1] → [Sand Filter] →
[Chemical Feed Pump] → [UV Sterilization] → [Clear Well] → [Dist Pump 1] → distribution
```

Treatment strategy: conventional sand filtration followed by combined chemical (sodium hypochlorite) and UV disinfection. Chlorine is dosed post-filtration, and UV provides the primary disinfection barrier. This order matters because chlorine at high concentrations can absorb UV wavelengths, so chlorine dosing rate (FIC-202) and UV effectiveness (RT-203) are implicitly coupled.

Intake Pump 2, Distribution Pump 2, and Booster Pump are parallel/backup paths shown as equipment but not in the main connection chain (the layout engine stacks them vertically within their stage).
